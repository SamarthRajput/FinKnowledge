run postgres image to postgres server using command 
docker run -p 5432:5432 -e POSTGRES_PASSWORD=mysecretpassword -d postgres
then the connection string will be
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Port=5432;Database=postgres;Username=postgres;Password=mysecretpassword"
  }
}

to create a dotnet project run the command 
dotnet new webapi -o <folder_name>
eg dotnet new webapi -o backend

to start the server we use command 
dotnet watch run

to migrate the database we need to run the command 
dotnet ef migrations add migration_Name
eg dotnet ef migrations add init

to update the database we need to run the command
dotnet ef database update 

models are database models 
Controllers are api controllers, we define our apis in Controllers
DTOs stands for data transfer object 
Mappers exist to convert data between different layers of your application — typically between your database entities (like Stock) and the objects you actually expose through your API (DTOs) — so those layers stay decoupled and don't directly depend on each other's internal structure.

We have mapped StockDto with the mapper and StockDto with the controllers
Interfaces dont really exists, Interfaces are only used to plugin with the repository
repository pattern means taking out all the database calls from controllers into function that are present in our repository
repository is existed only for database calls
